# 橙色底色来源与修改说明

## 一、橙色从哪里来？

### 1. 主来源：Gradio 主题 `primary_hue="orange"`

**位置**：`frontend/ui.py` 的 `launch_demo()`：

```python
demo.launch(
    ...
    theme=gr.themes.Soft(primary_hue="orange"),
    ...
)
```

**机制**：
- Gradio 的 `gr.themes.Soft(primary_hue="orange")` 会用**橙色**作为主题的“主色”，并生成大量 CSS 变量（如 `*primary_50` ~ `*primary_950`）。
- 这些变量被用在**输入框背景、边框、焦点高亮、滑块、块级容器**等地方，所以你会看到：
  - 第一步的「食材名称」「搜索补全」「涮煮时间」「份数」以及语音/识图等输入区域带**橙/黄底**或**橙色边框/高亮**。
- 主题变量名使用**下划线**，例如：
  - `input_background_fill` → 实际注入为 `--input_background_fill`
  - `block_background_fill` → `--block_background_fill`
  - 以及 `*primary_*` 等派生变量

因此：**橙色底色 = 主题的 primary 色（orange）被应用到表单/块级组件的背景和边框上**。

### 2. 我们已有的覆盖为什么可能不生效？

在 `assets/style.css` 里已经对第一步输入区做了：

```css
#page-step0 #ing-card-group,
#page-step0 #img-rec-group {
  --input-background-fill: var(--hotpot-white) !important;
  --background-fill-primary: var(--hotpot-white) !important;
  ...
}
```

以及直接对 `input`、`.gr-box` 等设 `background: var(--hotpot-white) !important`。

可能原因：
1. **变量名不一致**：Gradio 实际注入的是**下划线**形式（如 `--input_background_fill`），而我们写的是**连字符**（`--input-background-fill`），导致没有覆盖到主题变量。
2. **作用域**：主题变量可能写在 `:root` 或更高层，子元素用 `var(--input_background_fill)` 取的是主题值；若我们只在 `#ing-card-group` 上重写，需要保证变量名与 Gradio 完全一致才会生效。

## 二、推荐修改方式

### 方案 A：改主题主色（从根源去掉橙色）

在 **`frontend/ui.py`** 里把主题改成不用橙色的主色，例如中性色，这样表单区域就不会再带橙底：

```python
# 原来
theme=gr.themes.Soft(primary_hue="orange"),

# 改为（中性灰/石板色，表单不再有橙底）
theme=gr.themes.Soft(primary_hue="slate"),
# 或
theme=gr.themes.Soft(primary_hue="neutral"),
```

- **优点**：一处修改，所有由主题控制的输入/块背景都会变成中性色，不再偏橙。
- **注意**：主按钮等也会变成灰色系；若希望保留“橙色强调”，可在 CSS 里单独给需要强调的按钮设背景色（例如用 `#btn-confirm-add`、`#btn-next-visible` 等）。

### 方案 B：保留橙色主题，用 CSS 覆盖主题变量（精确去橙）

若必须保留 `primary_hue="orange"`（例如按钮等要保持橙色），只希望**第一步输入区**没有橙底，则需要在 CSS 里用 **Gradio 实际使用的变量名（下划线）** 覆盖。

在 **`assets/style.css`** 中，对第一步输入区增加/改为使用下划线变量名，例如：

```css
#page-step0 #ing-card-group,
#page-step0 #img-rec-group {
  /* Gradio 使用下划线命名 */
  --input_background_fill: var(--hotpot-white) !important;
  --block_background_fill: var(--hotpot-white) !important;
  /* 若存在连字符版本也一并覆盖 */
  --input-background-fill: var(--hotpot-white) !important;
  --background-fill-primary: var(--hotpot-white) !important;
  --background-fill-secondary: var(--hotpot-white) !important;
}
```

并保留对 `input`、`select`、`textarea`、`.gr-box`、`.gr-form` 等直接设 `background: var(--hotpot-white) !important`，这样即使变量名有出入，也能保证输入框是白底。

### 方案 C：用 Theme 的 .set() 在 Python 里把输入/块背景设为白

在 **`frontend/ui.py`** 里构造主题时，用 Gradio 的 `.set()` 把和输入/块背景相关的变量设为白色，例如：

```python
theme = gr.themes.Soft(primary_hue="orange").set(
    input_background_fill="#ffffff",
    block_background_fill="#ffffff",
)
demo.launch(..., theme=theme, ...)
```

这样是从主题层面统一改，不依赖 CSS 选择器；变量名需以 Gradio 文档/源码为准（一般为下划线形式）。

---

## 三、总结

| 来源 | 说明 |
|------|------|
| **根本原因** | `theme=gr.themes.Soft(primary_hue="orange")` 让主题用橙色作为 primary，并应用到输入框、块背景等。 |
| **变量命名** | Gradio 主题变量多为**下划线**（如 `input_background_fill`），不是连字符。 |
| **推荐** | 若不需要全局橙色：用**方案 A** 把 `primary_hue` 改为 `"slate"` 或 `"neutral"`；若只想去掉某块橙底且保留主题：用**方案 B** 在 CSS 里用下划线变量名 + 直接 `background` 覆盖。 |

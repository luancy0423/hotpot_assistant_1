# 尺寸与布局变量说明

项目中控制**尺寸**的代码已统一到 `assets/style.css` 的 **:root 尺寸变量表**，避免分散在 Gradio 参数、内联 HTML 与多处 CSS 中导致冲突。

## 一、尺寸变量表（:root）

所有关键尺寸仅在 `assets/style.css` 顶部定义，修改一处即可全局生效。

| 变量名 | 含义 | 典型用途 |

| `--app-breakpoint` | 媒体查询断点 | @media (min-width: 500px) 桌面端手机框 |
| `--app-frame-border` | 桌面端手机框边框粗细 | 模拟实体机边框 |
| `--app-frame-radius` | 桌面端手机框圆角 | 模拟实体机圆角 |
| `--card-margin` | 卡片水平外边距 | #ing-card-group、#plan-scroll-wrap 等 |
| `--card-padding` | 卡片内边距 | 输入区、卡片内容 |
| `--card-padding-lg` | 大卡片内边距 | 首页行动块 #hp-action-block |
| `--card-radius` | 卡片圆角（大） | 首页海报、输入区、按钮 |

## 二、使用约定

1. **禁止在 components.py 中写死尺寸**  
   不要使用 `style="width: 112px"`、`style="height: 100px"` 等。应使用类名（如 `.hp-poster`、`.status-box`），在 `assets/style.css` 中通过上述变量或类选择器定义尺寸。

2. **Gradio 组件尺寸**  
   - 布局占比：在 `frontend/ui.py` 中用 `gr.Column(scale=3)`、`gr.Row()` 等控制。
   - 具体像素（如图片高度）：仍可在 Python 中写 `height=180`，若希望统一管理，可在 CSS 中用选择器覆盖（如 `#boiling-image-input`）并引用变量。

3. **!important 使用**  
   仅在对「顶层容器」等易被 Gradio 主题覆盖的键（如 `max-width`、`padding-bottom`）保留必要 `!important`，其余尽量通过选择器优先级（如 `#page-step0 #ing-card-group`）控制，便于后续覆盖和调试。

## 三、修改流程示例

- **整体加宽手机框**：在 `assets/style.css` 的 `:root` 中把 `--app-max-width` 从 `450px` 改为 `480px`，主容器、底栏、抽屉会同步变化。
- **调整卡片间距**：修改 `--card-margin`、`--card-padding`。
- **调整「开始吃饭」按钮大小**：修改 `--btn-circle-size`。

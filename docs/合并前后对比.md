# 合并前后程序对比（基于 git 95f64df division → 3320c9b Merge）

## 合并前（commit 95f64df "division"）

### 架构
- **CSS**：全部写在 `frontend/ui.py` 里的 `_CSS` 字符串，**没有** `assets/style.css`。
- **状态**：用多个 `gr.State`（step_state、plan_data_state、ingredient_table_state 等），**没有** `frontend/state.py` 的 `AppState`。
- **组件**：从 `components` 引入 `basket_bar_html`、`ingredient_table_html`、`delete_selected_ingredient_row`；从 `parsers` 引入 `batch_add_ingredients`。

### 样式要点（与合并后差异）
- **整体容器**：`.gradio-container { background: #f7f5f2 }`（整机米/奶油色），固定 390×844 手机框。
- **步骤页底色**：统一浅米色  
  - `#page-step0 { background: #f3f0ec; }`  
  - `#page-step1 { background: #f3f0ec; min-height: 60vh; }`  
  - `#page-step2 { background: #f3f0ec; }`
- **Step2 方案区**：  
  - `#plan-scroll-wrap { background: white; margin: 12px; border-radius: 14px; padding: 16px 18px; box-shadow: 0 2px 14px rgba(0,0,0,.07); max-height: 44vh; overflow-y: auto; }`  
  - 带 scrollbar 样式（scrollbar-width: thin; scrollbar-color: #e0d8d0 #f5f0ea 等）。
- **卡片**：Step0/Step1 的 `#ing-card-group`、`#img-rec-group`、`.pref-acc` 等均为 **white**，步骤页底是 **#f3f0ec**。
- **步骤头**：`.shuai-step-bar { background: transparent }`，文字深色（#333 / #1a1a1a），**不是**红底金字。

---

## 合并后（当前）

- **CSS**：独立文件 `assets/style.css`，设计令牌（如 `--card-bg`、`--shuai-cell-bg` #EDEBDE）。
- **状态**：统一 `AppState`（`frontend/state.py`），含 timer 暂停/恢复、剔除食材等。
- **步骤头**：红底金字（渐变 + 金色数字）。
- **Step2**：若只给 `#plan-scroll-wrap` 米色而整页透明，会变成“只有一块米色卡片”，和合并前“整页 #f3f0ec + 中间白卡片”不一致。

---

## 为恢复 Step2 合并前观感做的修改

在 `assets/style.css` 中：

1. **Step2 整页**：使用合并前的浅米色 `#f3f0ec`（与合并前一致）。
2. **Step2 方案区**：`#plan-scroll-wrap` 使用 **白底** + 原 box-shadow/圆角/滚动条样式（与合并前一致）。

这样 Step2 的视觉效果与合并前一致：整页浅米 + 中间白色方案卡片。
